# 技术方案设计文档

技术栈：Android + Kotlin

---

## 一、项目概述

### 1.1 项目功能

本项目是一个多频道单列信息流展示应用，核心功能包括：

| 功能模块 | 功能描述 |
|---------|---------|
| 用户登录注册 | 支持用户名密码登录和注册 |
| 三频道信息流 | 精选、推荐、本地三个频道，通过 ViewPager2 切换 |
| 内容卡片 | 大图、小图、视频三种卡片类型，支持点赞、收藏 |
| 标签过滤 | 点击卡片标签进行过滤，跨频道同步（观察者模式） |
| 详情页 | 展示完整内容，视频播放控制，分享功能 |
| 视频播放 | 全局单例管理，支持 AndroidVideoCache 缓存播放 |
| AI 智能搜索 | DeepSeek API 对话，理解意图后检索数据库 |
| 用户中心 | 浏览历史、点赞列表、收藏列表的展示与跳转 |
| 页面动画 | Activity 切换动画、搜索提示滚动动画等 |

## 五、标签过滤模块（FilterManager）

文件：`FilterManager.kt`

**设计模式**：观察者模式

通过 `selectedTags` 集合存储选中的标签，订阅者通过 `addListener()` 注册监听。标签变化时通过 `notifyListeners()` 通知所有订阅者。

**主要订阅者**：
- **MainActivity**：更新顶部选中标签的显示
- **BaseChannelFragment**（三个频道实例）：重新加载并过滤数据

**触发方式**：点击 FeedAdapter 中卡片的标签 → `FilterManager.toggleTag(label)`

**过滤逻辑**（在 BaseChannelFragment 的 loadNextPage 中）：
- 有过滤标签时，加载全部数据 → `filter { it.label.any { label -> tags.contains(label) } }`
- 无过滤标签时，正常分页加载 `dao.getItemsSync(channelType, page * pageSize, pageSize)`

## 六、视频播放管理模块

文件：`util/VideoPlaybackManager.kt`

**设计模式**：全局单例

**核心设计原则**：任何时刻只允许一个视频处于播放状态，避免多个视频声音混杂和资源冲突。

### 6.1 核心方法

| 方法 | 功能 |
|-----|-----|
| `getProxyUrl(context, originalUrl)` | 获取带缓存的代理 URL |
| `isCached(context, url)` | 判断视频是否已缓存 |
| `play(itemId, videoView, playButton, coverView, videoUrl)` | 播放视频（自动暂停之前的视频） |
| `pauseCurrent()` | 暂停当前视频，显示播放按钮和封面 |
| `resumeCurrent()` | 恢复播放，隐藏封面 |
| `stop()` | 停止播放并清理状态 |
| `getCurrentPlayingId()` | 获取当前播放视频 ID |

### 6.2 视频缓存机制

使用 **AndroidVideoCache**（`com.danikula:videocache:2.7.1`），构建本地 HTTP 代理服务：

- **缓存目录**：`context.cacheDir`
- **最大缓存大小**：512 MB
- **最大缓存文件数**：20 个
- **代理 URL**：`http://127.0.0.1:<port>/<originalUrl>`

**工作流程**：
1. 首次播放：边下载边播放 → 写入本地缓存
2. 再次播放：直接从本地缓存读取 → 零网络消耗

### 6.3 BaseChannelFragment 中的视频智能暂停

RecyclerView 滚动时，检测当前播放视频是否仍在可见区域，若超出可见范围则自动调用 `VideoPlaybackManager.pauseCurrent()`。

## 七、AI 智能对话模块

### 7.1 架构概览

三层架构：UI 层（DialogActivity）→ 网络层（AiClient + AiService）→ 数据层（SearchService）

**API 服务**：DeepSeek API（`https://api.deepseek.com/v1/`）

**核心依赖**：
- `com.squareup.retrofit2:retrofit:2.9.0`
- `com.squareup.retrofit2:converter-gson:2.9.0`
- `com.squareup.okhttp3:okhttp:4.12.0`
- `com.squareup.okhttp3:logging-interceptor:4.12.0`

### 7.2 完整对话流程

1. **用户输入**：点击搜索框进入 DialogActivity，输入消息
2. **构建 Prompt**：AiClient.buildPrompt() 构建系统提示词（包含可用检索方法、标签说明、输出 JSON 格式）+ 用户消息
3. **API 请求**：DeepSeek API → model = "deepseek-chat"，Authorization: Bearer <API_KEY>
4. **响应解析**：尝试直接解析 JSON → 失败则从文本中提取 `{...}` → 仍失败则将全文作为纯文本回复
5. **数据库检索**：SearchService.executeSearch() 根据 AI 返回的 searchMethod 调用对应检索方法
6. **UI 展示**：显示 AI 纯文本消息（AI）或 AI 文本 + 搜索结果卡片（AI_WITH_SEARCH）
7. **点击卡片**：跳转到 DetailActivity 查看完整内容

### 7.3 Prompt 模板

```
系统指令：
- 你是智能助手，帮助用户通过标签检索广告内容
- 可用检索方法：searchByTag / searchByMultipleTags / searchByChannel / searchByKeyword
- 数据库标签：运动、旅行、科技、美食、美妆、家居 等
- 输出格式：{"reply": "...", "search_method": "...", "tags": ["..."]}
- 不需要检索时 search_method = "none"，tags = []

用户消息：<user_message>
```

### 7.4 SearchService 检索方法

文件：`dialog/SearchService.kt`

| 检索方法 | 功能 | 逻辑 |
|---------|-----|-----|
| `searchByTag(tag)` | 单标签匹配 | 全部频道数据 → 任一标签包含 tag |
| `searchByMultipleTags(tags)` | 多标签匹配 | 任一标签命中即返回 |
| `searchByChannel(channel)` | 按频道检索 | 直接查询该频道，最多 10 条 |
| `searchByKeyword(keyword)` | 关键词匹配 | 标题/描述/详情内容模糊匹配 |

## 八、页面 Activity 设计

### 8.1 LoginActivity（启动页）

**启动方式**：AndroidManifest.xml 中声明为 LAUNCHER

**核心功能**：
- 用户输入用户名和密码
- 点击登录：`UserManager.login()` → 成功则跳转 MainActivity
- 点击注册：`UserManager.register()` → 成功则跳转 MainActivity

### 8.2 MainActivity（主界面）

**布局结构**：顶部搜索框（带滚动提示文字） + 右上角三点菜单 → 三个频道按钮（精选/推荐/本地） → ViewPager2（三个 Channel Fragment）

**核心功能**：

| 功能 | 实现要点 |
|-----|---------|
| 频道切换 | ViewPager2 + 3 个独立 Fragment，按钮高亮联动 |
| 搜索提示滚动 | Handler + Runnable，每 3 秒切换一次搜索提示 |
| 点击搜索框 | 启动 DialogActivity，可传入当前提示文字 |
| 三点菜单 | PopupWindow 弹窗，点击进入用户中心 |
| 标签过滤监听 | 作为 FilterManager 订阅者，动态展示/移除选中标签 |

**Fragment 缓存**：`viewPager2.offscreenPageLimit = 2`，三个 Fragment 常驻内存，切换时不销毁不重建，保持浏览位置。

**数据初始化**：`onCreate` 中调用 `DataInitHelper.initTestData()`（挂起函数），完成后再设置 Adapter，确保数据就绪。

### 8.3 BaseChannelFragment + Channel_1/2/3

Channel_1/2/3 继承 BaseChannelFragment，仅通过构造函数传入不同的 channelType 参数（0/1/2），复用所有分页、刷新、滚动、过滤逻辑。

**核心功能**：

| 功能 | 实现要点 |
|-----|---------|
| 分页加载 | pageSize = 11，倒数第 3 条触发下一批 |
| 下拉刷新 | SwipeRefreshLayout → 清空数据重新加载 |
| 标签过滤 | FilterManager 监听 → 重新加载全量数据并过滤 |
| 详情页跳转 | 通过 Intent 传递所有字段（包含 like/star 状态），使用 ActivityResultLauncher 监听返回更新状态 |
| 视频智能暂停 | 滚动检测当前播放视频是否仍在可见区域 |

### 8.4 DetailActivity（详情页）

**数据来源**：从 Intent Extras 读取完整字段

**布局结构**：顶部标题 → 描述 → 图片/视频播放器 → 详情内容 → 标签 → 底部操作栏（点赞/收藏/分享）

**核心功能**：

| 功能 | 实现要点 |
|-----|---------|
| 自动播放 | 视频类型进入页面后自动播放（VideoView + 缓存代理 URL） |
| 视频控制 | 播放/暂停切换（点击视频区域 + 控制栏按钮） |
| 全屏切换 | 设置 FLAG_FULLSCREEN，视频尺寸 MATCH_PARENT |
| 静音切换 | AudioManager 设置 STREAM_MUSIC 音量 |
| 点赞/收藏 | 切换状态 → 更新数据库 → 更新 UI（颜色 + 数字） |
| 浏览历史 | 进入页面时自动调用 `UserManager.addBrowseHistory(itemId)` |
| 分享 | `Intent.ACTION_SEND` 分享标题和描述文本 |
| 返回动画 | 从右到左滑出（slide_in_left + slide_out_right） |

### 8.5 UserProfileActivity（用户中心）

**入口**：MainActivity 右上角三点菜单 → "个人中心"

**三个可折叠面板**（通过点击 header 切换内容区域 VISIBLE/GONE，箭头旋转动画）：

| 面板 | 数据来源 | 展示 |
|-----|---------|-----|
| 浏览历史 | User.browseHistory → dao.getItemsByIds() | RecyclerView + UserItemsAdapter |
| 我的点赞 | User.likeList → dao.getItemsByIds() | 同上 |
| 我的收藏 | User.starList → dao.getItemsByIds() | 同上 |

**登出**：调用 `UserManager.logout()` 清空当前用户，跳转回 LoginActivity

## 九、列表适配器设计

### 9.1 FeedAdapter（主信息流）

文件：`FeedAdapter.kt`

**设计**：继承 `ListAdapter<AdItem, RecyclerView.ViewHolder>`，内置 DiffUtil 差异对比，无需手动刷新全列表。

**4 种视图类型**：

| ViewType | 类型值 | 布局文件 | 说明 |
|---------|-------|---------|-----|
| TYPE_BIG | 0 | item_big_image.xml | 大图卡片，标题→图→标签→描述→操作栏 |
| TYPE_SMALL | 1 | item_small_image.xml | 小图卡片，左文右图布局 |
| TYPE_VIDEO | 2 | item_video.xml | 视频卡片，封面图 + 播放按钮 |
| TYPE_FOOTER | 3 | item_footer.xml | 底部"已加载全部"提示 |

**共同交互**：
- 点击卡片 → 跳转 DetailActivity（通过 Listener 回调到 Fragment）
- 点击标签 → `FilterManager.toggleTag(label)` 触发过滤
- 点赞按钮 → 切换状态 + 更新数据库 + 弹跳动画（scale_bounce）
- 收藏按钮 → 同上

**视频卡片交互**：点击播放按钮 → `VideoPlaybackManager.play(item.id, videoView, playBtn, coverView, videoUrl)`

### 9.2 MessageAdapter（对话消息）

文件：`dialog/MessageAdapter.kt`

**3 种视图类型**（由 MessageType 决定）：
- **USER**（右对齐）：气泡式文本消息
- **AI**（左对齐）：气泡式 AI 回复
- **AI_WITH_SEARCH**（左对齐）：气泡文本 + 下方多个搜索结果卡片（标题/图片/描述/标签，点击跳详情）

### 9.3 UserItemsAdapter（用户中心列表）

文件：`UserProfileActivity.kt`（内部类）

**布局**：`item_user_list.xml`，左图右文，最多显示 2 个标签。

## 十三、设计模式与架构亮点

### 13.1 设计模式总结

| 设计模式 | 应用位置 | 价值 |
|---------|---------|-----|
| **单例模式** | AppDatabase、UserManager、VideoPlaybackManager、FilterManager | 全局统一管理，避免资源冲突和重复创建 |
| **观察者模式** | FilterManager | 标签状态变化时自动通知所有订阅者刷新 |
| **基类继承** | BaseChannelFragment → Channel_1/2/3 | 三个频道复用分页、刷新、过滤、滚动逻辑 |
| **ListAdapter + DiffUtil** | FeedAdapter、MessageAdapter、UserItemsAdapter | 智能差异更新，避免全列表刷新 |
| **DAO 模式** | AdItemDao、UserDao | 数据访问与业务逻辑解耦 |
| **Repository 模式** | SearchService | 封装多种数据库检索方法供上层调用 |

### 13.2 数据流向设计

```
Room 数据库 (ad_items / users)
      ↓
DAO 接口 (AdItemDao / UserDao)  ← 异步查询（Dispatchers.IO）
      ↓
业务层管理器 (UserManager / FilterManager / VideoPlaybackManager)
      ↓
UI 层 (Activity / Fragment + RecyclerView + Adapter) ← 主线程更新（Dispatchers.Main）
      ↓
用户交互 (点击/滚动/输入) → 回调至业务层 → 更新数据库 → 循环
```

## 十四、核心数据交互链路

### 14.1 点赞状态同步（跨列表/详情页）

```

      ↓

      ↓

      ↓

### 14.2 浏览历史记录

```
点击卡片 → DetailActivity.onCreate()
      ↓

      ↓
history.remove(itemId) + history.add(0, itemId) （去重 + 最新优先）
      ↓
if (history.size > 50) history.removeLast() （保留最近 50 条）
      ↓
GlobalScope.launch(Dispatchers.IO) { userDao.updateUser(user) }
      ↓
UserProfileActivity.onResume() → 从数据库按 ID 列表查询 → 更新三个面板
```

### 14.3 AI 对话检索

```

      ↓

      ↓

      ↓

      ↓
SearchService.executeSearch(aiResponse.searchMethod, aiResponse.tags)
      ↓
searchByTag / searchByMultipleTags / searchByChannel / searchByKeyword
      ↓

      ↓

      ↓

      ↓
