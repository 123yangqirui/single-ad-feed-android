# 项目实现程度文档

## 1. 项目基本要求

1. 核心交互
  1. 实现单列广告信息流布局，支持流畅滚动与列表位置保持
  2. 广告卡片支持多样式：大图、小图、视频
  3. 顶部 Tab 支持切换频道（如：精选 / 电商 / 本地 ），切换时刷新数据
  4. 点击卡片进入详情页（图文/视频详情），返回保持原位置；
  5. 支持下拉刷新、上拉加载更多
2. 资源复用：
  1. 外流 cell 的复用
  2. 播放器资源的复用
  3. 缓存池的设计
3. 数据/状态同步： 
  1. 数据获取方式（本地 mock 还是走网络请求）
  2. 数据的管理（数据的生命周期，缓存方式）
  3. 数据、状态的传递（跨页面的数据如何传递和同步），比如：详情页和信息流的点评赞状态同步
  4. 视频播放器：暂停/续播、静音/音量控制。比如： 外流视频默认暂停状态，点击按钮进行播放/进入内流自动播放
4. 动画
  1. 页面切换动画
  2. 交互的动效，比如点赞彩蛋等
5. 埋点统计
  1. 支持模拟互动：本地状态管理，点赞、收藏、分享
  2. 且支持展示「曝光/点击」等统计信息（本地模拟 + 可视化展示）

## 3. 资源复用实现程度

### 3.1 列表 Cell 的复用

| 要求 | 实现程度 | 说明 |
|------|----------|------|
| RecyclerView 原生 View 回收 | ✅ 100% | 使用 `RecyclerView`，其内部维护 `RecycledViewPool`，自动管理已滚出屏幕的 View 复用。 |
| DiffUtil 局部刷新 | ✅ 100% | 继承 `ListAdapter`，通过 `DiffUtil.ItemCallback` 计算新旧列表差异，只对变更 item 执行局部刷新，避免 `notifyDataSetChanged()` 全量重绘。 |
| 三种类型独立缓存池 | ✅ 100% | `getItemViewType()` 区分 0/1/2 三类，`RecycledViewPool` 按 viewType 分别缓存，同类 item 直接复用，无需重新 inflate。 |
| 列表滚动位置保持 | ✅ 100% | `LinearLayoutManager` 自动保存 `onSaveInstanceState` 滚动位置，Fragment 重建后恢复。 |
| 标签 / 图片复用 | ✅ 100% | `bindLabels()` 使用 `removeAllViews + addView` 循环复用容器；`Glide` 内置三级缓存（内存 → 磁盘 → 网络），图片资源可跨列表共享。 |

### 3.3 缓存池的设计

| 要求 | 实现程度 | 说明 |
|------|----------|------|
| Video 视频缓存（核心） | ✅ 100% | 引入第三方库 **AndroidVideoCache**（`HttpProxyCacheServer`），在 `VideoPlaybackManager.getProxy()` 构建：设置缓存目录为 `context.cacheDir`、最大缓存 512MB、最多缓存 20 个文件。播放前将原始 URL 通过 `getProxyUrl()` 转换为本地代理 URL，由代理服务完成边下边播与二次读取本地缓存。 |
| 视频缓存状态判断 | ✅ 100% | 提供 `isCached(context, url)` 查询某个视频是否已完成磁盘缓存。 |
| 失败回退机制 | ✅ 100% | 代理 URL 播放失败时，`setOnErrorListener` 回退为直连原始 URL 再尝试。 |
| 图片缓存 | ✅ 100% | 通过 `Glide.with(context).load(url)` 加载，Glide 自带：（1）内存缓存 `LruResourceCache`；（2）磁盘缓存 `DiskLruCache`（策略 `AUTOMATIC`）；（3）网络请求层。重复 URL 图片零成本复用。 |
| RecyclerView View 缓存 | ✅ 100% | RecyclerView 内部 `RecycledViewPool` 按 viewType 保存废弃 ViewHolder，`onBindViewHolder` 复用已创建的 View，避免频繁 `LayoutInflater.inflate()`。 |
| 数据库查询结果缓存 | ✅ 部分 | Room 本身支持 `Flow` 观察者模式自动刷新，但项目当前以一次性同步查询为主；重复进入同一频道时，因 Fragment 被 `offscreenPageLimit` 常驻内存，`dataList` 仍保留上次分页结果，不重复触发 DB 查询。 |

## 7. 核心功能之外的额外功能实现

### 7.1 用户系统（登录 / 注册 / 登出）

**核心文件**：`LoginActivity.kt` + `dataprocess/UserManager.kt` + `dataprocess/UserDao.kt`

- **注册**：用户名唯一检查（`userDao.checkUsernameExists(username)`），创建新 User 记录。
- **登录**：用户名 + 密码校验，成功后写入 `UserManager.currentUser`。
- **登出**：`UserManager.logout()` 持久化当前用户状态并清空 `currentUser`。
- **入口**：应用启动 `AndroidManifest.xml` 中 `LoginActivity` 为启动页，登录/注册成功后跳转 `MainActivity`。
- **安全性说明**：密码明文存储（本地 Demo 场景），生产环境应使用加盐哈希。

### 7.2 用户中心（浏览历史 / 点赞列表 / 收藏列表）

**核心文件**：`UserProfileActivity.kt`

- 三栏可折叠展示：浏览历史、点赞列表、收藏列表。
- 折叠箭头带旋转动画，数据从 `User.browseHistory / likeList / starList` 取 ID，再由 `dao.getItemsByIds()` 查询完整详情。
- 每项可点击跳回 `DetailActivity` 查看详情。
- 右上角"个人中心"入口：`MainActivity.btnMore → PopupWindow → UserProfileActivity`。
- 提供一键登出功能并返回登录页。

### 7.3 标签筛选（FilterManager）

**核心文件**：`FilterManager.kt`

- 全局单例管理已选标签 `MutableSet<String>`，支持 `addTag / removeTag / toggleTag / clearTags`。
- 任何页面可通过 `addListener()` 订阅标签变更事件（观察者模式）。
- `MainActivity` 顶部实时展示当前选中标签，标签支持点击 × 删除。
- `FeedAdapter.bindLabels()` 中标签本身也可点击，点击后切换"加入筛选 / 取消筛选"状态（选中态用高亮颜色与背景区分）。
- `BaseChannelFragment` 订阅筛选事件，筛选生效时：
  - `dao.getAllItemsByChannel(channelType)` 查出该频道所有数据
  - `filter { item.label.any { tags.contains(it) } }` 按标签交集过滤
  - 不分页一次性展示过滤结果，清空筛选后回到正常分页

### 7.4 搜索框 + 提示词轮播

**核心文件**：`MainActivity.kt`（`searchHintText`、`btn_search`）

- 搜索栏位于顶部，显示动态轮播提示词（"找运动相关的广告"、"发现更多旅行资讯"等，3 秒一换，配合上下滑入/滑出动画）。
- 点击提示文字区域 → 进入自由输入的 `DialogActivity`。
- 点击搜索按钮 → 将当前提示词作为默认查询传入 `DialogActivity` 自动发送。
- 点击搜索框其他区域 → 与提示文字区域一致行为进入对话页。

### 7.5 AI 对话搜索（DialogActivity + AiClient + SearchService）

**核心文件**：`dialog/DialogActivity.kt`、`dialog/AiClient.kt`、`dialog/AiService.kt`、`dialog/AiResponse.kt`、`dialog/SearchService.kt`、`dialog/Message.kt`、`dialog/MessageAdapter.kt`

- **对话形式**：仿微信聊天界面，用户消息靠右、AI 消息靠左；支持滚动到底部、输入法回车发送。
- **AI 响应处理**：`AiClient.sendMessage(content, callback)` 调用大模型接口，返回 `AiResponse` 包含 `reply`（回复文本）、`searchMethod`（检索策略：`keyword` / `tag` / `none`）、`tags`（提取出的关键词列表）。
- **数据检索**：`SearchService.executeSearch(searchMethod, tags)` 根据策略：
  - `keyword`：在 title、desc 中模糊匹配；
  - `tag`：按标签集合交集匹配；
  - `none`：不检索，仅展示回复文本。
- **搜索结果卡片**：若有检索结果，消息类型 `AI_WITH_SEARCH`，下方以卡片形式展示匹配广告，点击即可跳转到详情页。
- **动画**：打开/关闭使用 `dialog_open_*` / `dialog_close_*` 动画。

### 7.6 视频本地代理缓存（AndroidVideoCache）

参见第 3.3 节。此能力使视频首次播放后，二次播放/在不同页面重复播放同一视频均可直接读取本地磁盘缓存，无需重复消耗网络流量。

### 7.7 更多菜单（PopupWindow）

**核心文件**：`MainActivity.kt:61-75, 204-234`

- 顶部右上角 `btnMore`（三点图标）→ 弹出 `PopupWindow`，包含"个人中心"入口。
- PopupWindow 支持点击外部关闭、半透明背景、入场偏移动画等交互细节。
